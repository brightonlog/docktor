## 8. 시연 시나리오

**1. 서버 환경 기동 (Web & Backend)**

포팅 매뉴얼의 **빌드 및 배포 가이드**에 따라 각 컨테이너와 서버를 실행합니다.

- **Step 1. 인프라 실행:** Redis, MQTT Broker를 먼저 기동합니다.
- **Step 2. AI 서버 실행:** Jetson Orin Nano와 통신할 Flask 서버를 실행합니다.
- **Step 3. API 서버 실행:** Spring Boot 웹 서버를 실행합니다.

  > Tip: 모든 서버가 기동된 후 docker ps 명령어로 상태가 Up인지 반드시 확인하십시오.
>

---

**2. 하드웨어 세팅 (Jetson Orin Nano)**

실제 구동 전 하드웨어의 안전 거리를 확보하고 네트워크 상태를 점검합니다.

- **기기 배치:** 결함 감지용 테스트 보드 전방 **1m 지점**에 젯슨 오린카를 평행하게 배치합니다.
- **네트워크 확인:** 웹 서버와 젯슨 오린 나노가 동일한 네트워크(또는 지정된 VPN)에 연결되어 있는지 확인합니다.
- **장비 점검:** 로봇카의 배터리 잔량 및 바퀴에 이물질이 없는지 육안으로 확인합니다.

---

**3. 동작 테스트 (Execution)**

API 호출을 통해 소프트웨어가 하드웨어 명령으로 정상 변환되는지 테스트합니다.

- **테스트 시나리오:** 전진 및 결함 검사 명령 수행 여부 확인
- **수행 방법:** Swagger UI 또는 Postman을 사용하여 아래 API를 호출합니다.
    - **Endpoint:** `POST http://{SERVER_IP}:8080/api/inspect/move-forward`
- **기대 결과:**
    1. Spring Boot에서 Flask 서버로 제어 신호 전달
    2. 젯슨 오린카가 전방으로 부드럽게 이동 및 결함 탐지
    3. 검사 종료 시 Spring Boot 웹 서버에서 검사 결과 확인 가능
