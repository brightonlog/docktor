# 🛠️ SafeDeck AI: 선박 결함 탐지 모델 학습 리포트

우리 팀의 'Safety-Beaver' 프로젝트는 Segmentation(테두리 따기)에서 시작해 
Detection(박스 찾기) 최적화를 거쳐, 최종적으로 목표치인 $mAP_{50}$ 0.74를 훌륭히 넘겼습니다!

---

## 📊 1. 전체 성능 변화 추이 (Total Performance)

| 단계 | 방식 | $mAP_{50}$ (Box) | $mIoU$ | 주요 성과 및 특징 |
|:---:|:---:|:---:|:---:|:---|
| **2차 시도** | Segmentation | 0.5060 | 0.4170 | 세그멘테이션 도입, 소형 객체(Pinhole) 탐지 한계 |
| **4차 시도** | Detection V4 | 0.7164 | **0.8515** | Bbox 방식 전환, 데이터셋 1.2만 장 확대 |
| **V5** | Detection V5 | 0.6995 | - | MLOps(MLflow) 도입, Scratch 데이터 부족(9장) |
| **V6 (Final)** | **Detection V6** | **0.7972** | - | **목표 달성!** Scratch mAP 7배 향상, 데이터 불균형 해소 |

---

## 🔍 2. 단계별 상세 분석

### [2차 시도] 기초 다지기: Segmentation
- **핵심 지표**: $mAP_{50}$ 0.506 / $mIoU$ 0.417
- **분석**: 'Sagging'은 잘 잡지만(IoU 0.93), 'Pinhole'(0.05)이나 'Crack'(0.01) 같은 미세한 결함은 거의 못 잡는 아이였어요.

### [4차 시도] 전략 변경: Detection 최적화
- **핵심 지표**: $mAP_{50}$ 0.716 (**+21%p** ↑) / $mIoU$ 0.851 (**+43%p** ↑)
- **변화**: 테두리 대신 박스로 찾는 방식(Detection)으로 바꿔서 속도와 정확도를 동시에 잡았어요. Pinhole 데이터를 4,725장 추가 확보한 게 신의 한 수!

### [V6] 목표 달성: 데이터 증강의 힘
- **핵심 지표**: $mAP_{50}$ **0.797** (**V5 대비 +9.8%p** ↑)
- **성공 요인**: 9장이던 Scratch 데이터를 AI Hub 자료를 활용해 **1,637장**으로 늘렸어요. (182배 증가!)
- **결과**: Scratch $mAP_{50}$가 0.116에서 **0.812**로 **7배(696%)**나 껑충 뛰었습니다.

## 🚀 4. 최종 결론 및 다음 단계
1. **성공**: $mAP_{50}$ 0.7972로 당초 목표(0.74+)를 **약 107% 초과 달성**했습니다.
2. **최적화**: 이제 이 모델을 젯슨 오린 나노에 올리기 위해 **TensorRT(.engine)** 변환 작업을 진행할 예정입니다.
3. **과제**: 성능이 상대적으로 낮은 Pinhole과 Peeling에 대해서는 추후 7차 시도에서 보강이 필요해요.
