## 📅 [3차 시도] 인스턴스 세그멘테이션 (Instance Segmentation)


![outputs1](./outputs/3rd_trials/outputs1.jpg)
* **학습 데이터:** 선박 도장 결함 데이터 (균열, 용접 손상, 스크래치 등)
    * 전체 데이터셋: 12,324장
    * 학습 데이터 (Train): 9,859장
    * 검증 데이터 (Val): 2,465장
* **학습 방법:** SAM(Segment Anything Model) + YOLOv8n-seg (Polygon 기반)
* **핵심 개선:**
* 기존의 네모난 박스 데이터를 SAM AI를 사용하여 **다각형(Polygon) 형태**로 업그레이드함
* 단순 탐지를 넘어 결함의 실제 테두리를 따라가는 분할 학습 실시함 


### 모델 평가

| 지표 (Metric) | 수치 (Value) | 의미 |
| --- | --- | --- |
| **Mask mAP50** | **0.500 (50.0%)** | 결함의 위치와 종류를 정확히 맞춘 확률 |
| **mIoU (Mean IoU)** | **0.4170 (41.7%)** | 실제 결함과 AI가 색칠한 영역이 겹치는 정도 |

### 클래스별 상세 성적표

| --- | --- | --- | --- |

### 3. 결과 분석

1. * `Sagging`은 **98.2%**라는 놀라운 성적을 보였지만, `Pinhole`은 **2.0%**에 불과함.
*  **면적이 넓고 테두리가 뚜렷한 결함**은 비교적 잘 잡으나, **'점처럼 작거나 색이 흐릿한 결함**은 잘 잡지 못하고 있음


2. **mAP50(50%) vs mIoU(41.7%)의 간격:**
* 탐지 능력(mAP)보다 색칠 능력(IoU)이 약 **8.3%p** 낮음.
* 즉, AI가 결함을 찾긴 하는데, 그 테두리를 실제 모양대로 따는 것에는 한계가 있음


3. **데이터 불균형의 영향:**
* 사진이 400장 이상이었던 `WeldingDamage`와 `Scratch`는 성적이 좋았지만, 100장 내외였던 `Pinhole` 등은 성적이 낮음

## 📅 [4차 시도] 객체 탐지 최적화 및 핀홀 데이터 보강 (Detection V4)

* **학습 방법:** YOLOv8n (Bounding Box 기반으로 복귀)
* **학습 데이터:** 선박 도장 결함 데이터 (핵심 결함 7종 중심)
* **전체 데이터셋:** 12,324장 (핀홀 데이터 4,725장 집중 보강)
* **학습 데이터 (Train):** 9,859장
* **검증 데이터 (Val):** 2,465장


* **핵심 개선 및 전략:**
* **Bbox 방식 회귀:** 세그멘테이션의 높은 연산량 대신, 실시간 탐지 속도와 작은 객체(핀홀) 인식 효율이 높은 **바운딩 박스** 방식으로 전환함
* **데이터 밸런싱:** 3차 시도에서 가장 취약했던 **Pinhole(핀홀)** 데이터를 대폭 추가하여 소형 결함 탐지 성능을 보강함
* **학습 파라미터 최적화:** 학습률을 **0.02**로 상향하여 대규모 데이터셋에서 모델이 더 과감하게 특징을 학습하도록 유도함

### 4차 시도 기대 효과 및 분석 포인트

1. **핀홀(Pinhole) 탐지율 개선:**
* 3차 시도에서 2.0%에 불과했던 핀홀 mAP를 데이터 보강과 Bbox 방식을 통해 유의미한 수준으로 끌어올리는 것이 이번 시험의 최대 목표임


2. **학습률 상향()의 영향:**
* 높아진 학습률이 1만 장의 대규모 데이터셋에서 손실 함수(Loss)를 얼마나 빠르게 수렴시키는지, 그리고 오버피팅 없이 안정적으로 학습되는지 모니터링함


3. **실무 효율성 검증:**
* 세그멘테이션 대비 빨라진 추론 속도를 확인하고, 선박 검사 현장에서 사용자가 결함을 즉각적으로 식별하기에 바운딩 박스 형태가 얼마나 더 직관적인지 평가함
